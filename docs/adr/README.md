# Architecture Decision Records (ADR)

Use ADRs to capture meaningful architectural choices and tradeoffs.

## Naming
- `NNNN-short-title.md`
- Example: `0001-adopt-nextjs-and-lambda.md`

## Required ADR Sections
- Context
- Decision
- Alternatives considered
- Consequences

## When ADR is Required
- technology/runtime choice
- auth/session model changes
- database model changes that affect invariants
- infrastructure strategy changes
